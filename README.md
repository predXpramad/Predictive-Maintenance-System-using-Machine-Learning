# 🔧 Predictive Maintenance System

> **Binary classification system to predict industrial machine failures  
> using the AI4I 2020 Predictive Maintenance Dataset.**

---

## 📌 Project Overview

Unplanned machine downtime costs manufacturing industries billions of dollars annually.
This project builds an end-to-end **Predictive Maintenance System** that analyses
real-time sensor data to flag machines likely to fail — enabling proactive maintenance
before a breakdown occurs.

The system consists of:

| Component | Technology | Purpose |
|-----------|-----------|---------|
| `train.py` | Scikit-learn / XGBoost | Model training & evaluation |
| `main.py` | FastAPI | REST API for serving predictions |
| `app.py` | Streamlit | Interactive dashboard UI |
| `model.pkl` | Joblib | Saved best-performing model |

---

## 🎯 Problem Statement

Given six machine sensor readings, predict whether the machine will **fail** (1) or
**operate normally** (0).

The dataset is highly imbalanced (96.6% No Failure / 3.4% Failure), which makes
precision and recall on the minority class critical metrics.

## 🗄️ Dataset

| Property | Value |
|----------|-------|
| Source | [UCI ML Repository – AI4I 2020](https://archive.ics.uci.edu/dataset/601) |
| Rows | 10,000 |
| Features used | 6 (after preprocessing) |
| Target | `Machine failure` (binary) |
| Failure rate | ~3.4% |

### Feature Engineering

| Step | Details |
|------|---------|
| Dropped | `UDI`, `Product ID` (identifiers) |
| Dropped | `TWF`, `HDF`, `PWF`, `OSF`, `RNF` (data leakage – failure sub-types) |
| Encoded | `Type`: L→0, M→1, H→2 (ordinal label encoding) |
| Imbalance | SMOTE applied on training data only |

## 🚀 How to Run

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Place the dataset

Copy `ai4i2020.csv` into the project folder.

### 3. Train the model

```bash
python train.py
```

This will:
- Train Random Forest and XGBoost classifiers
- Print a side-by-side comparison table
- Save the best model as `model.pkl`
- Generate `feature_importance_*.png` and `confusion_matrix_*.png`

### 4. Start the FastAPI backend

```bash
python main.py
# OR
uvicorn main:app --reload --port 8000
```

API docs available at: http://localhost:8000/docs

### 5. Launch the Streamlit dashboard

```bash
streamlit run app.py
```

Dashboard available at: http://localhost:8501

## 🔌 API Reference

### `GET /`  — Health Check

```bash
curl http://localhost:8000/
```

**Response:**
```json
{
  "status": "ok",
  "model": "XGBoost",
  "features": ["Type", "Air temperature [K]", ...]
}
```

---

### `POST /predict` — Predict Machine Failure

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "Type": 1,
    "Air temperature [K]": 298.5,
    "Process temperature [K]": 308.6,
    "Rotational speed [rpm]": 1551,
    "Torque [Nm]": 42.8,
    "Tool wear [min]": 108
  }'
```

**Response:**
```json
{
  "prediction": 0,
  "label": "✅ No Failure",
  "probability_failure": 0.0342,
  "model_used": "XGBoost"
}
```

**Field Reference:**

| Field | Type | Description |
|-------|------|-------------|
| `Type` | int | 0=Low, 1=Medium, 2=High quality |
| `Air temperature [K]` | float | Air temp in Kelvin |
| `Process temperature [K]` | float | Process temp in Kelvin |
| `Rotational speed [rpm]` | int | Spindle speed |
| `Torque [Nm]` | float | Applied torque |
| `Tool wear [min]` | int | Cumulative tool use time |

---

## 💡 Why XGBoost Wins

1. **Gradient Boosting** – Sequentially corrects residual errors; captures subtle failure patterns that Random Forest misses.
2. **Built-in Regularisation (L1/L2)** – Prevents overfitting on the small failure class.
3. `scale_pos_weight` – Natively up-weights the minority class during training.
4. **Column & row subsampling** – Produces diverse trees with lower variance.
5. **Fine-grained learning rate** – Small `learning_rate=0.05` allows precise decision boundary adjustment.
